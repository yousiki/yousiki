# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Repository purpose

This repository powers the GitHub profile for `yousiki`. `README.md` embeds generated SVG assets rather than rendering content directly in Markdown.

## Development commands

- Regenerate the profile card and social badges locally:
  - `node scripts/render-profile.mjs`
- There is no package manifest or formal test/lint setup in this repository.
- Focused verification is asset-based rather than test-based:
  - `node scripts/render-profile.mjs`
  - `git diff -- assets/profile.svg assets/badges/*.svg`

## Architecture

### Source of truth

- `scripts/render-profile.mjs` is the core generator. It fetches live tokscale badge data and the contribution snake SVG, then renders:
  - `assets/profile.svg`
  - `assets/badges/*.svg`
- Prefer editing `scripts/render-profile.mjs` rather than hand-editing generated SVG output.

### How the profile is assembled

- `README.md` is intentionally minimal: it displays `assets/profile.svg` and links the generated badge SVGs.
- The main profile card is a single inline SVG composition generated by `renderProfile(...)` in `scripts/render-profile.mjs`.
- Tokscale metrics come from `https://tokscale.ai/api/badge/yousiki/svg?metric=...` and are parsed out of returned SVG badge markup.
- The contribution snake is generated as a local intermediate SVG at `dist/snake.svg` during the main workflow run, then inlined into `assets/profile.svg`.

### Automation

- `.github/workflows/render-profile.yaml` is the main refresh workflow. It runs on:
  - a 6-hour cron schedule
  - manual dispatch
  - pushes to `main` that change `scripts/render-profile.mjs` or the workflow itself
- That workflow uses `Platane/snk` to generate `dist/snake.svg`, then runs `node scripts/render-profile.mjs`, and commits `assets/profile.svg` plus `assets/badges/*.svg` if they changed.

## Working conventions

- Generated assets live under `assets/`; source logic lives under `scripts/`.
- If you change profile layout, badge text, or metric formatting, validate by regenerating locally and reviewing the SVG diff.
- If the snake section appears missing locally, check whether `dist/snake.svg` exists before running `node scripts/render-profile.mjs`; the script falls back gracefully when that intermediate file is absent.
